# K3s Homelab Cluster

> A learning project for Kubernetes using K3s on bare metal

## Cluster Architecture

| Hostname   | Role           | CPU/RAM         | IP              |
|------------|----------------|-----------------|-----------------|
| adama      | control plane  | i5-7200U, 8GB   | 192.168.1.20    |
| boomer     | worker         | N150, 16GB      | 192.168.1.21    |
| apollo     | worker         | N150, 16GB      | 192.168.1.22    |
| starbuck   | worker         | N150, 16GB      | 192.168.1.23    |

---

## Installation Steps

### 1. SSH into Each Node and Install K3s

```bash
ssh bsg@adama
curl -sfL https://get.k3s.io | sh -
```

On the control plane (adama), get the node token:
```bash
sudo cat /var/lib/rancher/k3s/server/node-token
```

**On each worker node (boomer, apollo, starbuck), join the cluster:**
```bash
ssh bsg@boomer
curl -sfL https://get.k3s.io | K3S_URL=https://192.168.1.20:6443 K3S_TOKEN=<NODE_TOKEN> sh -s - agent

ssh bsg@apollo
curl -sfL https://get.k3s.io | K3S_URL=https://192.168.1.20:6443 K3S_TOKEN=<NODE_TOKEN> sh -s - agent

ssh bsg@starbuck
curl -sfL https://get.k3s.io | K3S_URL=https://192.168.1.20:6443 K3S_TOKEN=<NODE_TOKEN> sh -s - agent
```
Replace `<NODE_TOKEN>` with the value from the previous step.

---

### 2. Verify the Cluster

On the control plane node (adama), run:
```bash
kubectl get nodes
```
You should see all nodes listed.

Label the worker nodes:
```bash
kubectl label node boomer node-role.kubernetes.io/worker=worker
kubectl label node apollo node-role.kubernetes.io/worker=worker
kubectl label node starbuck node-role.kubernetes.io/worker=worker
```

---

### 3. Configure Remote Kubeconfig (Recommended Way)

**On adama:**
```bash
sudo cp /etc/rancher/k3s/k3s.yaml ~/k3s-remote.yaml
sudo chown $USER:$USER ~/k3s-remote.yaml
chmod 600 ~/k3s-remote.yaml
sed -i 's/127.0.0.1/192.168.1.20/' ~/k3s-remote.yaml
```

**On your local machine:**
```bash
scp bsg@192.168.1.20:~/k3s-remote.yaml ~/k3s.yaml
export KUBECONFIG=~/k3s.yaml
echo 'export KUBECONFIG=~/k3s.yaml' >> ~/.bashrc
kubectl get nodes
```

**If you get a TLS error:**  
Edit `~/k3s.yaml` and add under `server:`:
```yaml
    insecure-skip-tls-verify: true
```
and comment out or remove the `certificate-authority-data:` line.
